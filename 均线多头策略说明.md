# 均线多头策略使用说明

## 问题排查

### 问题：每日选股没有均线多头策略选出的股票

### 可能原因和解决方法

---

## 1. 数据库中未添加策略记录

**检查方法**：
```sql
USE StockAnalysisSystem;
SELECT * FROM `Strategy` WHERE `StrategyType` = 'MultiMovingAverage';
```

**解决方法**：
```bash
mysql -u root -p StockAnalysisSystem < Scripts/insert_multi_ma_strategy.sql
```

---

## 2. 数据库未添加 VolumeMA120 字段

**检查方法**：
```sql
USE StockAnalysisSystem;
SHOW COLUMNS FROM `StockDailyIndicator` LIKE 'VolumeMA120';
```

**解决方法**：
```bash
mysql -u root -p StockAnalysisSystem < Scripts/migration_add_volume_ma120.sql
```

---

## 3. 历史数据不足（最常见原因）

**检查方法**：
```sql
USE StockAnalysisSystem;
SELECT
    COUNT(DISTINCT `StockId`) as total_stocks,
    COUNT(CASE WHEN `VolumeMA120` IS NOT NULL THEN 1 END) as stocks_with_ma120
FROM `StockDailyIndicator`;
```

**结果判断**：
- 如果 `stocks_with_ma120` 很少或为 0 → 历史数据不足 120 天
- 需要导入更长的历史日线数据（至少 120 天以上）

**具体检查某只股票**：
```sql
SELECT
    `TradeDate`,
    `Volume`,
    `VolumeMA120`,
    `MA5`,
    `MA10`
FROM `StockDailyIndicator`
WHERE `StockId` = '某个股票ID'
ORDER BY `TradeDate` DESC
LIMIT 15;
```

---

## 4. 策略条件过于严格

**策略条件**：
- 5日线 > 10日线（均线多头）
- 收盘价 > 5日线（价格突破）
- 最近 5 天至少有 2 天量能 > 120日量能均线 × 3 倍（量能放大）

**临时调整参数（放宽条件）**：
```sql
USE StockAnalysisSystem;

UPDATE `Strategy`
SET `Parameters` = '{"ShortPeriod":5,"MediumPeriod":10,"VolumeMaPeriod":120,"VolumeMultiplier":2.0,"CheckDays":10,"RequiredExpansionDays":1}'
WHERE `StrategyType` = 'MultiMovingAverage';
```

参数说明：
- `VolumeMultiplier`: 3.0 → 2.0（降低量能倍数要求）
- `CheckDays`: 5 → 10（增加检查天数）
- `RequiredExpansionDays`: 2 → 1（降低放量天数要求）

---

## 完整执行步骤

### 步骤 1：添加数据库字段
```bash
mysql -u root -p StockAnalysisSystem < Scripts/migration_add_volume_ma120.sql
```

### 步骤 2：插入策略记录
```bash
mysql -u root -p StockAnalysisSystem < Scripts/insert_multi_ma_strategy.sql
```

### 步骤 3：重新计算指标
在系统中执行一次数据更新，让系统计算并保存 VolumeMA120 指标

### 步骤 4：测试选股
在每日选股界面选择"均线多头策略"进行选股

---

## 策略参数说明

| 参数 | 默认值 | 说明 |
|-----|--------|------|
| ShortPeriod | 5 | 短期均线周期（MA5） |
| MediumPeriod | 10 | 中期均线周期（MA10） |
| VolumeMaPeriod | 120 | 量能均线周期 |
| VolumeMultiplier | 3.0 | 量能放大倍数阈值（量能 > 120日均量 × N 倍） |
| CheckDays | 5 | 检查最近 N 天的量能情况 |
| RequiredExpansionDays | 2 | 要求至少 N 天满足量能放大条件 |

## 信号示例

当策略发出买入信号时，理由格式：
```
均线多头(MA5>MA10), 收盘价>12.35, 最近5天有3天量能>3倍120日均量
```

## 常见问题

### Q1: 为什么大多数股票的 VolumeMA120 是 NULL？

**A**: 因为历史数据不足 120 天。需要导入至少 120 天以上的历史日线数据。

### Q2: 数据已充足但仍不选股？

**A**: 可以执行以下 SQL 查看具体数据：
```sql
SELECT
    d.`StockCode`,
    d.`TradeDate`,
    d.`ClosePrice`,
    i.`MA5`,
    i.`MA10`,
    d.`Volume`,
    i.`VolumeMA120`,
    CASE
        WHEN i.`MA5` > i.`MA10` AND d.`ClosePrice` > i.`MA5` THEN '均线多头且突破'
        ELSE '不满足'
    END as condition1,
    CASE
        WHEN d.`Volume` > i.`VolumeMA120` * 3 THEN '满足'
        ELSE '不满足'
    END as condition2
FROM `StockDailyData` d
LEFT JOIN `StockDailyIndicator` i ON d.`StockID` = i.`StockId` AND d.`TradeDate` = i.`TradeDate`
WHERE d.`StockCode` IN (
    SELECT DISTINCT `StockCode`
    FROM `StockDailyData`
    ORDER BY `TradeDate` DESC
    LIMIT 10
)
ORDER BY d.`TradeDate` DESC
LIMIT 50;
```

这个查询可以帮您分析具体哪些条件不满足。
